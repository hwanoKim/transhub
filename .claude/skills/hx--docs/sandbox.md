# Sandbox (실험적 아이디어)

## 사용 시점

- 아직 검증되지 않은 아이디어
- 프로토타입 문서화
- 실험적 접근 방식 기록
- 나중에 정식 문서로 승격 가능

## 저장 위치

```
.docs/references/sandbox/
├── ref-sandbox-index.md
└── {YYYYMM}/
    └── {YYYYMMDD}-{title}.md
```

## 파일명 형식

```
{YYYYMMDD}-{title}.md
```

예시: `20260123-new-ui-concept.md`

## Sandbox vs Research

| 항목 | Sandbox | Research |
|------|---------|----------|
| 목적 | 아이디어 기록, 실험 | 정보 수집, 조사 |
| 검증 | 미검증 | 검증된 정보 |
| 출처 | 본인 아이디어 | 외부 소스 |
| 승격 | 정식 문서로 이동 가능 | 보통 그대로 유지 |

## 상태 관리

| 상태 | 설명 |
|------|------|
| 초안 | 처음 작성, 아직 검토 안함 |
| 검토중 | 검증/검토 진행 중 |
| 승격됨 | 정식 문서로 이동 (architecture, components 등) |
| 폐기 | 더 이상 유효하지 않음 |

## 승격 절차

아이디어가 검증되어 정식 채택 시:

1. 적절한 폴더로 문서 이동 (architecture, components, dev-guides)
2. 브레드크럼 업데이트
3. 해당 인덱스에 추가
4. sandbox 인덱스에서 제거 또는 "승격됨" 표시
5. 원본에 승격 기록:
   ```markdown
   **상태**: 승격됨 → [[새위치문서]]
   ```
